## What Is DataCREAM Agentic System?

DataCREAM Agentic System is built on top of the DataCREAM data analysis agent.

It allows users to describe their business scenario in natural language. The system then investigates the scenario, clarifies the real analytical requirements, identifies entities and dimensions, generates data analysis tasks, creates a page plan, and builds a business-facing application.

The goal is simple:

```text
Business Scenario
        ↓
Data Analysis Requirement
        ↓
Business Data Model
        ↓
Agent Workflow
        ↓
Business Application
```

Most data analysis tools start with data.

DataCREAM Agentic System starts with business context.

It helps users who understand their business but may not understand data analysis, algorithms, data modeling, or application development.

## Typical Workflow

```text
Create Agent
    ↓
Structure Business
    ↓
Generate Page Plan
    ↓
Generate Webpage
    ↓
Upload Data
    ↓
Analyze Data
    ↓
Use Results in Business Application
```

### Step 1: Create Agent

The user starts a new business agent.

They select a business area and provide a business background.

### Step 2: Structure Business

The system investigates the business scenario and structures the requirement.

It identifies possible entities, dimensions, data sources, analysis tasks, and application goals.

### Step 3: Generate Page Plan

The system creates a reviewable page plan.

The page plan defines how the final application should support the business process.

### Step 4: Generate Webpage

The system builds a business-facing web application based on the plan.

### Step 5: Upload Data

Users upload relevant data such as spreadsheets, documents, transaction records, customer data, product data, supplier data, or business reports.

### Step 6: Analyze Data

DataCREAM executes the analysis tasks using clustering, ranking, embedding, analysis, and matching.

### Step 7: Use Results

Users review and use the results inside the generated business application.

The goal is not only to see insights, but to use insights in real decisions.

## Why Not Just Use an LLM?

LLMs are powerful for language understanding and generation.

However, enterprise data analysis requires more than language reasoning.

It requires:

* Structured entity modeling
* Multi-dimensional representation
* Persistent knowledge accumulation
* Reusable analytical operators
* Explainable analysis logic
* Business workflow integration
* Controlled and repeatable execution

DataCREAM Agentic System uses LLMs where they are useful, especially for scenario understanding, requirement clarification, and explanation.

But the analytical foundation is DataCREAM’s structured knowledge and analysis layer.
